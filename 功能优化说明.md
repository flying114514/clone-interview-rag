# 功能优化说明

本文档说明本次新增和优化的三项功能，涉及后端接口修正、前端历史页增强和面试页自动保存三个模块。

---

## 1. 简历库增加状态筛选与排序

### 功能说明

在简历库（历史记录）页面新增两个筛选控件，支持对简历列表进行本地筛选和排序：

- **状态筛选**：按分析状态过滤，支持「全部」「待分析」「分析中」「已完成」「失败」五种状态
- **排序方式**：支持按上传时间（升序/降序）、AI评分（升序/降序）、文件名升序排列

无分数记录的简历在评分排序时统一排在末尾，保证排序结果稳定。

### 涉及文件

| 文件 | 变更说明 |
|------|---------|
| `app/src/main/java/interview/guide/modules/resume/service/ResumeHistoryService.java` | `getAllResumes()` 方法改用 MapStruct 映射器构造 `ResumeListItemDTO`，确保 `analyzeStatus`、`analyzeError`、`storageUrl` 等字段正确返回 |
| `frontend/src/pages/HistoryPage.tsx` | 新增 `StatusFilter`、`SortOption` 类型；`statusFilter`、`sortBy` 状态；`resolveAnalyzeStatus`、`getAnalyzeStatusText`、`getAnalyzeStatusStyles`、`sortResumes` 工具函数；筛选/排序 UI 控件及对应的过滤/排序逻辑 |

### 交互细节

- 筛选和排序均在浏览器本地执行，无需请求后端
- 状态语义兼容旧数据：`analyzeStatus` 明确时直接使用；缺失但有 `latestScore` 时视为 `COMPLETED`；均无时视为 `PENDING`
- 筛选结果为空时显示友好提示：「未找到匹配的简历，请调整搜索关键词或筛选条件后重试」
- 「分析状态」列展示状态徽章（色点+文案），失败状态鼠标悬停显示错误详情

## 2. 上传成功后自动高亮并定位新简历

### 功能说明

上传简历成功并跳转到简历库页面后，系统会自动将新上传的简历滚动到屏幕中央并高亮显示，同时在文件名旁显示「新上传」标签，4 秒后自动淡出。

### 涉及文件

| 文件 | 变更说明 |
|------|---------|
| `frontend/src/pages/HistoryPage.tsx` | 接入 `useLocation`、`useNavigate`；新增 `highlightResumeId`、`pendingHighlightId`、`rowRefs`、`highlightTimerRef` 状态和 Refs；通过 `location.state.newResumeId` 接收跳转时传入的简历 ID；实现分步重置（清空搜索词 → 重置筛选 → 重置排序 → 执行定位）的等待链；定位后立即用 `replace: true` 清空 `location.state` 防止刷新重复触发 |

### 交互细节

- 上传跳转已在 `frontend/src/App.tsx` 中通过 `navigate('/history', { state: { newResumeId } })` 传递 ID，本页消费该状态
- 如果当前筛选/搜索条件会将目标简历隐藏，会自动重置为显示全部、最新上传优先
- 高亮通过 Tailwind 背景色 `bg-primary-50 dark:bg-primary-900/20` 实现
- 刷新页面或返回历史页不会重复高亮（`location.state` 在首次消费后立即清理）

## 3. 面试页面增加自动保存状态提示

### 功能说明

在模拟面试页面，回答过程中答案会按 1.2 秒防抖自动暂存到服务器，后台静默保存。输入框上方实时显示保存状态：

- **自动保存中**：显示「自动保存中...」
- **已保存**：显示「已自动保存于 XX:XX」
- **保存失败**：显示「自动保存失败，内容仍保留在当前页面」

### 涉及文件

| 文件 | 变更说明 |
|------|---------|
| `frontend/src/pages/InterviewPage.tsx` | 新增 `SaveStatus` 类型；`saveStatus`、`lastSavedAt` 状态；`autoSaveTimerRef`、`lastSavedAnswerRef`、`saveRequestSeqRef`、`currentSessionIdRef`、`currentQuestionIndexRef`、`currentAnswerRef`、`isSubmittingRef` 等 Refs；`clearAutoSaveTimer`、`syncDraftToSession`、`saveDraftAnswer` 工具函数；防抖自动保存 Effect（依赖 `answer`、`currentQuestion`、`isSubmitting`、`session`、`stage`）；切换题目时重置保存状态；`handleSubmitAnswer` 中清除定时器并同步草稿到 session；提前交卷时同步当前草稿 |
| `frontend/src/components/InterviewChatPanel.tsx` | 新增 `saveStatus`、`lastSavedAt` props；`saveStatusText` 计算属性生成状态文案；输入区上方增加状态提示区域 |

### 自动保存触发条件

仅在以下条件全部满足时触发自动保存：

1. `stage === 'interview'` 且存在有效的 `session` 和 `currentQuestion`
2. 当前不在提交中（`!isSubmitting`）
3. 输入内容非空且与上次保存内容不同

### 上下文安全（防竞态）

- 使用递增 `requestSeq` 标记每次保存请求，自动丢弃过期响应
- 响应返回时校验 `sessionId`、`questionIndex`、`answerSnapshot` 均与当前上下文一致
- 恢复未完成面试时，将已恢复的 `userAnswer` 同步到 `lastSavedAnswerRef`，避免刚恢复就立即重复保存同样内容
- 提交答案成功后立即同步草稿到前端 session，保证页面刷新/返回时能正确恢复最新内容

### 边界情况

- 输入空白时只显示错误状态（不显示 null placeholder）
- 提交失败后输入内容保留，自动保存仍可继续工作
- 切到下一题后自动重置保存状态，不会在新题上显示上一题的保存时间
- 提前交卷时也会将当前草稿同步到 session

## 4. 附带修复：后端简历列表接口字段补全

### 问题描述

`ResumeHistoryService.getAllResumes()` 中手动构造 `ResumeListItemDTO` 时遗漏了 `analyzeStatus`、`analyzeError`、`storageUrl` 三个字段，导致前端状态筛选无数据可用。

### 修复方式

将手动构造改为调用 `resumeMapper.toListItemDTO(...)`，该方法已完整映射 DTO 的全部 11 个字段。

### 涉及文件

| 文件 | 变更说明 |
|------|---------|
| `app/src/main/java/interview/guide/modules/resume/service/ResumeHistoryService.java` | `getAllResumes()` 中将 `new ResumeListItemDTO(...)` 替换为 `resumeMapper.toListItemDTO(resume, latestScore, lastAnalyzedAt, interviewCount)` |
